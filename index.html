HTML Document Layouts

Introduction

There are five ways to create multi-sectional layouts:

  1. HTML tables
  2. CSS frameworks
  3. CSS float property
  4. CSS flexbox
  5. CSS grid

Tables are not recommended. The purpose of <table> is to display tabular data.

CSS frameworks

CSS frameworks provide a basic structure for designing consistent solutions to tackle common issues across front-end web development. They provide generic functionality which can be overridden for specific scenarios and applications. This greatly decreases the time needed to start creating applications and websites.

The packages usually consist of HTML, CSS, and JavaScript files and provide a grid, browser incompatibility solutions, responsive media queries, uniform styling across tooltips, buttons, forms etc, and a consistent set of fonts and icons.

Popular frameworks include:

  • Bootstrap
  • Foundation
  • Materialize
  • Pure
  • UIKit

CSS float

It is common to do entire web layouts using the float property. Float is easy to learn - you just need to remember how the float and clear properties work. But floating elements are tied to the document flow which can reduce flexibility.

The float property specifies how an element should float. It can have one of four vales: left, right, none, or inherit. The default is none. At its simplest, float can be used to wrap text around images.

            img { float: right; }

The clear property specifies what elements can float beside the cleared element and on which side. It can have five possible values: none, left, right, both, inherit. The most common way to use the clear property is after you have used a float property on an element. When clearing floats, you should match the clear to the float: If an element is floated to the left, then you should clear to the left. Your floated element will continue to float, but the cleared element will appear below it on the web page.

            div { clear: left; }

If an element is taller than the element containing it, and it is floated, it will "overflow" outside of its container. To fix this you can use overflow: auto; in the containing element. This works well if you have control over the padding and margins. But a safer version to use is:

            .clearfix::after {
                content: "";
                clear: both;
                display: table;
            }

CSS flexbox

CSS3 introduced Flexible Boxes, or flexbox, to create page layouts for a dynamic UI. It is a layout mode that arranges elements in a predictable way for different screen sizes and browsers. All popular modern browsers support flexbox. Adding display: flex; turns an element into a flex container. Then to align children of that element into rows or columns use flex-direction. Possible values of flex-direction are row, column, row-reverse, column-reverse. The default value is row.

Items in a flex-box can be horixontally aligned using justify-content. Possible values are flex-start, flex-end, center, space-between, and space-around. Items can be vertically aligned using align-items. Values include flex-start, flex-end, center, stretch, and baseline.

CSS grid

CSS Grid helps you easily build complex web designs. It works by turning an HTML element into a grid container with rows and columns within which you place child elements. Turn any HTML element into a grid container with display: grid;. To define the structure of the grid add columns with grid-template-columns and rows with grid-template-rows

            grid-template-rows: 50px 50px 50px;
            grid-template-columns: 25% 25% 50%;

Each parameter represents a row or column and defines its width (or height). You can use px, em, %, fr, or auto. fr sets the width to a fraction of the available space and auto sets it to the size of its content. If you have several parameters with the same value you can use the repeat function: grid-template-rows: repeat(3, 50px); which creats 3 rows each of 50px.

You can create gaps between the rows and columns using grid_row-gap and grid-column-gap or the shorthand grid-gap where the first parameter is the row gap and the second the column gap. If only one parameter is given it will create gaps between all rows and columns.

You can set items in the grid to span more than one row or column using grid-row: 2/3; and grid-column: 1/4; This would create an area for the item covering rows 2 and columns 1 to 3. The numbers represent the edges of the grid squares, not the squares themselves. A shortcut for this is using grid-area: horizontal line to start at / vertical line to start at / horizontal line to end at / vertical line to end at; which in the above example would be grid-area: 2/1/3/4;.

You can align items horizontally and vertically using justify-self and align-self or align all items using justify-items and align-items. Possible values are start, end, and center.

You can create grids within grids.
